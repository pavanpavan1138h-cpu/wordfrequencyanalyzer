Algorithm Comparison

Naive O(n²)

Naive Method

Uses nested loops to compare every word against every other word. Simple but slow for large datasets.

Time: O(n²) Space: O(n)
—
Hashing O(n)

Hashing Method

Uses a hash table with chaining for collisions. Near-constant time per operation on average.

Time: O(n) avg Space: O(n)
—
Dictionary O(n)

Dictionary Method

Uses a key-value dictionary (associative array). Direct key lookup for O(1) average insert and search.

Time: O(n) avg Space: O(k)
—

1. Naive Method

Time: O(n²)

Uses nested loops to iterate through every word and compare it against all previously seen words. Conceptually simple, but becomes extremely slow as input size grows.

⚡

2. Hashing Method

Time: O(n) avg

Employs a hash table with separate chaining for collision resolution. Words are hashed to indices, allowing near-constant time insertion and lookup.

📖

3. Dictionary Method

Time: O(n) avg

Uses a key-value associative array mapping words directly to counts. It achieves O(1) average-case lookups using JavaScript's native Map object.

⚙️

System Architecture & Data Flow

Engineered for high-throughput text ingestion, our architecture leverages JavaScript Streams and the native File API for non-blocking file processing. Raw data flows into a dynamic in-memory buffer, undergoes sanitization via regex-driven parsers, and is seamlessly routed to the selected algorithmic core. The resulting metrics are instantaneously rendered through an interactive visualization pipeline.

🛠️

Tech Stack & Processing

Built with HTML5, CSS3, and Vanilla JS. Uses Chart.js for visualizations and the native File API & Streams for chunk-by-chunk processing to effortlessly handle files exceeding 100MB.

Built as a 4th Semester Engineering Lab project for Design Analysis & Algorithms.
